# Security Policy

## Supported Versions

| Version | Supported |
| ------- | --------- |
| Latest  | Yes       |
| Older   | Best effort |

We always support the latest release. Older versions may receive security patches if feasible, but upgrading to the latest version is recommended.

## Reporting a Vulnerability

If you discover a security vulnerability, please report it privately by emailing **zoom-poker-jockey@duck.com**.

Please do **not** open a public GitHub issue for security vulnerabilities.

### What to expect

- You will receive an acknowledgement within 48 hours.
- We will investigate and provide an initial assessment as soon as possible.
- Once a fix is ready, we will coordinate disclosure with you before publishing.

### What to include

- A description of the vulnerability and its potential impact.
- Steps to reproduce or a proof of concept, if possible.
- The version(s) affected.
